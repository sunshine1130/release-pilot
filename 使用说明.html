🔝 页面顶部操作栏

搜索框 🔍 搜索需求名称、链接ID或分支名…

支持实时过滤：输入后自动在所有需求中搜索，匹配需求名称、需求链接、涉及项目分支名、一同发版分支名。右侧 × 按钮清除搜索内容。

导出 按钮

点击将当前所有数据（项目列表、需求列表、分支缓存、上线分支名）导出为 todoList-发版需求-YYYYMMDD-HHmmss.json 文件并下载到本地。

导入 按钮

点击选择之前导出的 .json 文件导入数据。导入后会覆盖当前全部数据并更新页面显示，同时清空编辑 / 新增中的状态。

获取分支 按钮

手动触发从本地 Git 仓库同步所有项目的最新分支列表。需要先启动 node server.js 本地服务器（端口 1949），且项目已配置本地路径。同步结果弹窗展示每个项目的分支列表，点击分支名可复制到剪贴板。

自动同步：页面首次打开时会自动后台同步（每天最多一次），同步期间顶部显示进度条。

+ 新增需求 按钮

在待发版表格顶部新增一行空白编辑表单，填写后保存即可创建新需求。

💾 数据备份提醒弹窗

每天首次打开页面时自动弹出，提醒用户备份数据——因为浏览器 localStorage 可能被清理。

  • 导出保存 — 执行导出操作并关闭弹窗，同时记录今日已提醒
  • 暂不保存 — 跳过本次备份，记录今日已提醒，今日内不再弹出

📊 分支同步进度条

自动后台获取分支时显示，位于顶部操作栏下方。显示当前同步进度（如 3/5）和进度条动画。同步完成后约 0.8 秒自动隐藏。

触发条件：

  • 有项目配置了本地路径
  • 本地服务器 node server.js 正在运行
  • 今天尚未执行过自动同步

🗂️ 项目管理面板

点击 ▶ 项目管理 展开 / 折叠。用于维护所有项目的元信息。

添加项目行

字段说明
项目名称必填，项目的唯一标识名称。后续创建需求时可从下拉选择。颜色标签按项目添加顺序自动分配（Material Design 色板）
MR 地址可选，GitLab 项目的 Merge Request 基础 URL（如 https://gitlab.com/group/repo）。配置后标签加粗边框，可点击跳转
本地路径可选，项目在本机的 Git 仓库路径。配置后可通过服务器获取该项目的最新分支
添加将项目添加到列表。重复名称会有提示

项目标签颜色规则

颜色按项目在列表中添加的先后顺序分配，而非随机。依次为：红 → 绿 → 蓝 → 橙 → 紫 → 青 → 黄 → 粉 → 靛 → 黄绿 → 深橙 → 浅蓝 → 棕（12 色循环）。保证了 7 个项目各不同色。

配置了 MR 地址的项目标签为实心色块 + 白字，否则为浅色底 + 深色字，一眼可区分。

项目公用上线分支名行

字段说明
如 master_launched输入所有项目共用的上线目标分支名（全局默认值）
保存上线分支保存到 localStorage 并更新所有已渲染的分支链接。清空输入框保存可删除

项目标签列表

每个项目以彩色标签展示，鼠标悬停显示操作按钮：

  • MR — 跳转到 GitLab 项目的 Merge Request 列表页
  • NB — 跳转到 GitLab 新建分支页，分支名自动生成为 master-MMDD（当天月日），基于 master 创建
  • 📁 — 鼠标悬停显示该项目配置的本地路径
  • ✏️ — 编辑项目名称、MR 地址、本地路径
  • × — 从列表中删除该项目
注意：删除项目不会影响已有需求中已关联的该项目记录，但会从下拉列表中移除。

⏳ 待发版表格

展示所有状态为新建和待发版的需求，默认展开。

表格列

列名说明
需求名称粗体显示需求名
状态新建 或 待发版
需求链接可点击跳转到需求详情页，超长 URL 自动截断显示
涉及项目及分支名 彩色项目名标签 + 绿色分支名标签 + 📋 复制按钮 + 📝 备注按钮（仅当有备注时显示，橙色可点击查看）。

操作按钮说明

  • 编辑 — 该行切换为编辑模式，可修改需求名称、链接、项目分支、一同发版分支、分支备注
  • 设为待发版 — 将新建需求转为待发版状态
  • 发版 — 将需求转为发版中状态。若未设置公用上线分支名，弹出"填写各项目上线目标分支"弹窗
  • 回退新建 — 将待发版需求退回新建状态
  • 删 — 确认后删除该需求

编辑模式下新增的按钮

  • 📂 — 查看该项目所有分支，点击分支名自动填充到输入框
  • 📝 — 编辑该分支的备注（弹窗中输入，支持 SQL / DDL / 注意事项等）
  • × — 移除该项目行
  • + 添加分支 / + 添加一同发版分支 — 新增一行

🚀 发版中表格

展示所有发版中的需求。有数据时自动展开。

表头右侧 全部发版 按钮

仅在发版中有数据时显示，点击一键将所有发版中的需求标记为已发版并记录当前时间。

项目摘要

表头右侧以彩色标签汇总展示当前所有发版中需求涉及的去重项目。有 MR 地址的项目可点击跳转 GitLab — 源分支取各需求的上线分支名，目标分支为 master。

分支名点击目标分支优先级

  1. 待发版表格始终为 uat
  2. 发版中表格：公用上线分支 > 本条目的自定义目标分支（targetBranches）> 无
  3. 若均未设置，buildMrUrl 最终兜底为 master_launched

操作按钮

  • 标记已发版 — 将需求转为已发版，更新发版时间
  • 恢复待发版 — 将需求退回到待发版，清除发版时间
  • 删 — 确认后删除
发版中和已发版不支持编辑，但分支备注仍可查看。

✅ 已发版表格

展示所有已发版的需求。

额外列

相比前两个表格多了发版时间列，显示格式为 YYYY-MM-DD HH:mm:ss。

操作按钮

  • 恢复待发版 — 退回待发版状态
  • 删 — 确认后删除

📝 上线分支填写弹窗

在未设置公用上线分支名时点击发版触发。列出当前需求涉及的所有项目，每行包含：

  • 彩色项目名标签
  • → 箭头
  • 目标分支输入框（默认为空，需手动填写，按 Enter 快速确认）

智能跳过

若发版中已有其他条目的同一项目填写过目标分支，该弹窗自动跳过该项目。全部跳过则直接流转到发版中，不弹窗。

确认发版

  • 所有输入框必填，空框标红提示
  • 确认后每条需求保存 targetBranches = { "项目A": "uat", "项目B": "release" }
  • 同时合并其他发版中已有的目标分支

交互

  • 按 Esc 关闭
  • 点击遮罩黑色区域关闭
  • 点击取消关闭并取消本次发版操作

⚠️ 确认删除弹窗

点击删按钮时弹出，显示被删除的需求名称。

  • 确定删除 — 执行删除，从 items 数组中移除，刷新页面
  • 取消 或按 Esc 或点击遮罩 — 放弃删除

📝 分支备注功能

为每个分支维护备注信息（如 SQL 脚本、DDL 变更、注意事项等），方便后端发版时参考。

查看模式（非编辑状态）

  • 有备注：分支名旁显示橙色 📝 图标，点击弹出只读弹窗查看备注内容
  • 无备注：不显示任何图标，保持界面简洁
  • 弹窗中显示项目名和分支名，textarea 为只读，底部仅有关闭按钮

编辑模式

  • 每个项目分支行都有 📝 可编辑按钮，始终显示
  • 有备注时按钮高亮橙色，无备注时灰色
  • 点击弹出可编辑弹窗，输入备注后保存
  • 备注随需求数据一起保存到 localStorage 和导出 JSON 中

弹窗交互

  • 支持 Esc 关闭、点击遮罩关闭
  • 保存后自动刷新列表并 toast 提示
  • 数据结构：{ name, branch, notes }

数据兼容

旧数据（无 notes 字段）完全兼容，会自动降级为无备注状态。

📂 分支查看弹窗

两种入口：

  1. 点击顶部获取分支按钮 — 展示所有配置了本地路径的项目分支
  2. 编辑模式下点击 📂 按钮 — 仅展示对应项目的分支，点击分支名自动填充到输入框

弹窗内容

  • 项目名 + 分支数量 + 搜索框（支持实时过滤分支）
  • 分支列表：分支名（蓝色可点击）+ 最近提交日期 + 提交者
  • 点击分支名：编辑模式下填充到输入框并 toast 提示；全局查看模式下复制到剪贴板
  • 已发版的项目分支会标灰并标记"已发版"

数据来源

  • 缓存优先：如果 branchesCache 中已有该项目数据，直接展示
  • 实时获取：缓存不存在时请求 http://localhost:1949/api/branches，结果自动存入缓存
分支数据已过滤：不显示 origin、origin/ 开头的远程引用，以及 master / test / dev / main / release / staging / hotfix / uat 等公共分支。

💿 数据导出 / 导入

导出格式

JSON 文件，包含以下字段：

字段内容
projectNames项目列表 [{name, mr, localPath}]
items需求列表 [{id, name, link, status, projects, syncBranches, releaseTime, targetBranches}]，其中 projects 和 syncBranches 每项含 {name/ project, branch, notes}
branchesCache分支缓存 {项目名: {branches:[], timestamp}}
launchBranch公用上线分支名

导入行为

  • 完全覆盖当前数据（项目、需求、缓存、上线分支）
  • 同步写入 localStorage
  • 清理编辑 / 新增中的状态（editingIds、addingItems）
  • 更新项目管理面板和上线分支输入框
  • 重新渲染所有表格

自动加载

页面初始化时优先加载同目录下的 toDo.json 文件（如存在），否则从 localStorage 加载。

🖥️ 本地服务器 node server.js

获取分支功能依赖本地 HTTP 服务器（端口 1949），用于在浏览器中调用 git for-each-ref 命令。

启动方式

cd d:\发版小助手
node server.js

核心 API

路径说明
GET /api/git-user获取当前 Git 用户名，用于分支过滤
GET /api/branches?path=<路径>&user=<用户名>获取指定仓库的分支列表

分支过滤逻辑

  1. 使用 git for-each-ref refs/remotes/origin/ refs/heads/ --sort=-authordate --count=300 一次性获取
  2. 过滤公共分支：master、test、dev、main、release、staging、hotfix、uat
  3. 过滤 origin 和 origin/ 远程引用
  4. 过滤超过 1 个月的旧分支
  5. 同名分支去重（保留最新的）
如果启动失败，请确保已安装 Node.js 且项目路径下有 .git 目录。

🖥️ 无 Node 环境的替代方案

如果无法安装 Node.js 或不方便启动本地服务器，以下功能仍可正常使用，只是缺少部分自动化能力。

完全可用的功能

  • 需求管理：新建、编辑、删除、状态流转（新建 → 待发版 → 发版中 → 已发版）
  • 项目管理：维护项目名称、MR 地址、本地路径
  • 分支备注：为每个分支添加 SQL、DDL 等备注信息，支持查看和编辑
  • MR 链接跳转：点击项目名 / 分支名跳转到 GitLab 创建 MR 页面（目标分支自动填充）
  • NB 按钮：跳转到 GitLab 新建分支页，分支名自动生成为 master-MMDD
  • 数据导出 / 导入：导出 JSON 文件备份，导入恢复
  • 搜索过滤：搜索需求名称、链接、分支名
  • 公用上线分支名：全局设置和保存
  • 上线分支填写弹窗：按项目自定义目标分支

受影响的功能

功能影响替代方案
获取分支 按钮点击后提示"请先启动服务器" 手动从 GitLab 网页查看分支，复制后粘贴到输入框
自动后台同步分支 不执行，不会显示进度条 无影响，仅不自动缓存分支列表
编辑模式 📂 查看分支 提示"该项目未配置本地路径"或"请先启动服务器" 手动输入分支名或使用导出的数据中已缓存的分支

分支名的手动处理方式

在编辑模式下，直接在分支名输入框中手动输入分支名即可。输入框支持：

  • 直接打字输入任意分支名
  • 空格自动被过滤，保持分支名无空格
  • 鼠标悬停在输入框上可看到已输入的内容

关于分支缓存

如果之前使用过 Node 服务器获取过分支，数据已缓存在浏览器 localStorage 中。导入带有 branchesCache 的导出文件后，之前缓存的分支列表仍可查看（📂 按钮可用）。

总结：没有 Node 环境的唯一实际影响是无法自动获取 Git 分支列表。所有其他核心功能完全独立运行，不受影响。
