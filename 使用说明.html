需求发版管理 - 使用说明

一个前端版本发布流程管理工具，帮助团队跟踪需求的发版状态、管理项目分支、快速跳转创建 MR。

一、快速开始

文件结构

文件说明
todo-release.html主页面（HTML 模板）
todo-release.js业务逻辑（Vue 3 组件）
todo-release.css样式文件
vue.global.jsVue 3 运行库
server.jsNode.js 本地分支服务
toDo.json数据文件（可选，优先级高于 localStorage）

所有文件放在同一目录下，直接用浏览器打开 todo-release.html 即可使用。无需构建工具。

数据存储优先级

  1. 同目录下的 toDo.json（如果存在，每次打开自动加载）
  2. 浏览器 localStorage（自动保存，关闭浏览器不丢失）
  3. 手动 导出/导入 JSON 文件（推荐定期备份）

二、需求管理

需求状态流转

每条需求有 4 种状态，按以下流程流转：

新建 → 待发版 → 发版中 → 已发版

  • 新建：刚创建的需求。点击"编辑"可修改信息，"设为待发版"进入下一状态。
  • 待发版：已确认待上线的需求。点击"发版"开始发版流程，支持编辑。
  • 发版中：正在发布的需求。记录发版时间，不支持编辑。可"标记已发版"或"恢复待发版"。
  • 已发版：发版完成的需求。记录发版时间，可"恢复待发版"。

新增/编辑需求

  1. 点击 "+ 新增需求" 按钮，在表格顶部出现编辑行。
  2. 填写需求名称（必填）和需求链接（可选）。
  3. 添加涉及项目及分支名：选择项目名、填写分支名。可添加多行，使用 📂 按钮查看该项目所有分支。
  4. 添加一同发版分支：填写依赖的同步分支信息（如第三方组件分支）。只有分支名非空时才会保存。
  5. 每个分支行可添加备注（📝），记录 SQL 脚本、DDL 变更、注意事项等。
  6. 点击"保存"完成新增。新增的需求会以高亮动画显示。

点击单条需求的"编辑"按钮可修改已有需求（仅在新建/待发版状态下可编辑）。

搜索与过滤

顶部搜索框支持按以下内容实时过滤：

  • 需求名称
  • 需求链接
  • 分支名称

点击搜索框右侧 ✕ 按钮或按 Escape 清除搜索。

三、发版流程

单条发版

  1. 确认需求处于 待发版 状态。
  2. 点击"发版"按钮。
  3. 如果已设置全局上线分支：直接转入发版中。
  4. 如果未设置全局上线分支：弹出窗口，为每个涉及的项目填写目标分支名（即要合并到 master 的上线分支）。
  5. 填写完成后点"确认发版"，需求状态变为 发版中。

目标分支会保存在该需求上，后续发版中的项目按钮会据此生成正确的 MR 链接。

发版中的项目 MR

在 "发版中" 分组的表头右侧，会显示当前发版涉及的项目按钮：

  • 点击项目按钮，直接跳转创建 MR（源: 上线分支 → 目标: master）。
  • 源分支的确定顺序：全局上线分支 > 发版时填写的项目目标分支 > master_launched。

全部发版

  • 发版中的需求列表旁有"全部发版"按钮，点击将所有发版中的需求标记为已发版。
  • 已发版的需求自动记录发版完成时间。

分支点击 MR 跳转

在需求列表中，点击分支名会自动复制分支名到剪贴板。如果该项目配置了 MR 地址，还会在新标签页打开 MR 创建页面：

需求状态MR 目标分支
新建test
待发版uat
发版中全局上线分支 > 项目目标分支 > null
已发版全局上线分支 > 项目目标分支 > null

四、项目管理

展开/折叠

点击 "项目管理" 区域可展开或折叠项目配置面板。

添加项目

在项目管理区域底部填写：

  • 项目名称（必填）
  • MR 地址（可选，填写 GitLab 项目根地址或 MR 页面地址，用于自动生成 MR 链接）
  • 本地路径（可选，填写项目在本机的绝对路径，用于获取分支列表）

按 Enter 或点击"添加"按钮。添加后项目会出现在标签列表中。

编辑/删除项目

每个项目标签右侧有 ✏️（编辑）和 ×（删除）按钮：

  • 点击 ✏️ 进入编辑模式，修改后点"保存"。
  • 点击 × 直接删除该项目。

项目快捷操作

每个项目标签旁的功能按钮：

按钮功能
MR跳转到项目的 Merge Request 列表页
NB跳转到新建分支页面（分支名自动为 master_MMDD，基于 master）
📁显示该项目配置的本地路径（鼠标悬停查看）

项目公用上线分支名

在项目管理区域底部设置全局上线分支名（如 master_launched），用于：

  • 发版中的项目按钮生成 MR（源: 此分支 → 目标: master）
  • 分支点击跳转 MR 时的目标分支回退值

按 Enter 或点击"保存上线分支"。

项目下拉选择

在新增/编辑需求时，输入项目名会自动弹出已维护的项目下拉列表：

  • 输入文字实时过滤匹配项目
  • 点击选项或上下键 + Enter 选择
  • 失去焦点后自动关闭

五、分支管理

前置条件：需要启动本地 Node.js 分支查询服务。
  1. 项目必须在"项目管理"中配置了本地路径
  2. 在终端执行 node server.js 启动服务（端口 1949）
  3. 服务启动后会向 GitLab API 请求当前用户的用户名，用于过滤个人分支

单项目分支查看

在新增/编辑需求的每个分支行，点击 📂 按钮：

  • 查看该项目所有远程分支（过滤掉 origin/ 开头）
  • 在弹窗中搜索过滤分支名
  • 点击分支名自动填充到当前分支输入框
  • 已被发版过的分支会以灰色标识（released-branch）

全部项目分支

点击顶部 "获取分支" 按钮：

  • 为所有配置了本地路径的项目获取最近分支列表
  • 结果以弹窗展示，按项目分组
  • 支持全局搜索过滤
  • 点击分支名直接复制到剪贴板

分支缓存

  • 获取分支后自动缓存到 localStorage，下次打开直接使用缓存
  • 每天首次打开页面时，自动后台静默刷新全部分支（当天已刷新则跳过）
  • 顶部有进度条显示刷新状态

六、备注功能

为分支添加备注

  • 在编辑模式下，每个分支行有 📝 按钮，点击打开备注编辑弹窗。
  • 备注内容可记录 SQL 脚本、DDL 变更、注意事项等。
  • 有备注的分支行 📝 按钮会高亮显示（has-notes 样式）。
  • 在查看模式下，如果有备注则显示 📝 按钮，点击可查看但不可编辑。

右键菜单

在待发版和新建需求的分支行上点击右键：

  • 📝 添加/编辑备注
  • 🗑️ 清除备注（仅当已有备注时显示）

七、导入导出

导出

  • 点击顶部 "导出" 按钮。
  • 生成 JSON 文件：todoList-发版需求-时间戳.json
  • 包含完整数据：项目列表、需求列表、分支缓存、上线分支设置。

导入

  • 点击顶部 "导入" 按钮，选择之前导出的 JSON 文件。
  • 导入会覆盖当前所有数据并自动保存。
  • 导入后自动清除编辑状态和新增行。

每日备份提醒

每天首次打开页面时，弹出备份提醒窗口：

  • "导出保存" — 自动导出 JSON 文件并关闭提醒
  • "暂不保存" — 关闭提醒（下次打开不再弹出当天）
